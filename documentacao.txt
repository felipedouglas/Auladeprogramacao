DOCUMENTACAO DO SISTEMA ACADEMICO

1. OBJETIVO DO PROJETO

Este projeto e um prototipo simples de um sistema academico desenvolvido em
Python com base nos conceitos de orientacao a objetos.

O sistema representa alunos, professores, disciplinas e uma estrutura central
responsavel por cadastrar e relacionar essas informacoes.


2. ARQUIVOS DO PROJETO

O codigo foi dividido nos seguintes arquivos:

- Pessoa.py
  Contem a classe Pessoa, que representa uma pessoa generica do sistema.

- Aluno.py
  Contem a classe Aluno, que herda da classe Pessoa.

- Professor.py
  Contem a classe Professor, que tambem herda da classe Pessoa.

- Disciplina.py
  Contem a classe Disciplina, responsavel por armazenar dados da disciplina,
  seu professor e seus alunos matriculados.

- Sistema_academico.py
  Contem a classe SistemaAcademico e a funcao main, que executa o exemplo
  principal do programa.

- app.py
  Contem a aplicacao web em Flask, com rotas para painel, alunos,
  professores, disciplinas, matriculas e relatorios.

- db.py
  Contem as funcoes de conexao com banco de dados. O sistema tenta usar MySQL
  e, se ele nao estiver disponivel, utiliza um banco SQLite local para permitir
  o uso da interface.

- relatorios.py
  Contem as funcoes responsaveis por gerar relatorios nos formatos JSON e PDF.

- schema.sql
  Contem os comandos SQL para criar o banco MySQL, as tabelas e os dados
  iniciais.

- templates/
  Pasta com as paginas HTML da interface web.

- static/styles.css
  Arquivo de estilos visuais da interface web.


3. CONCEITOS DE ORIENTACAO A OBJETOS UTILIZADOS

3.1 Classe

Classe e um modelo usado para criar objetos. Neste projeto foram criadas as
classes Pessoa, Aluno, Professor, Disciplina e SistemaAcademico.

3.2 Objeto

Objeto e uma instancia de uma classe. Por exemplo:

professor = Professor(...)
aluno_1 = Aluno(...)
disciplina = Disciplina(...)

3.3 Heranca

A heranca permite que uma classe reaproveite atributos e metodos de outra.
Neste projeto, Aluno e Professor herdam de Pessoa.

Pessoa possui os atributos nome e cpf. Como Aluno e Professor tambem sao
pessoas, essas classes reutilizam esses atributos por meio da heranca.

3.4 Encapsulamento

Cada classe organiza seus proprios dados e comportamentos. Por exemplo, a
classe Disciplina controla sua lista de alunos e o professor responsavel.

3.5 Associacao entre objetos

O sistema tambem usa associacao entre objetos:

- Um Aluno pode estar associado a varias disciplinas.
- Um Professor pode estar associado a varias disciplinas.
- Uma Disciplina possui um professor e uma lista de alunos.


4. CLASSE PESSOA

Arquivo: Pessoa.py

A classe Pessoa representa uma pessoa generica do sistema.

Atributos:

- nome
  Armazena o nome da pessoa.

- cpf
  Armazena o CPF da pessoa.

Metodos:

- __init__(self, nome, cpf)
  Metodo construtor da classe. Ele recebe nome e cpf e armazena esses valores
  no objeto.

- __str__(self)
  Retorna uma representacao textual da pessoa, exibindo nome e CPF.

Exemplo de uso:

pessoa = Pessoa("Joao", "123.456.789-00")
print(pessoa)


5. CLASSE ALUNO

Arquivo: Aluno.py

A classe Aluno representa um aluno do sistema academico.

Heranca:

Aluno herda da classe Pessoa. Por isso, todo aluno possui nome e cpf.

Atributos:

- nome
  Herdado de Pessoa.

- cpf
  Herdado de Pessoa.

- matricula
  Armazena o numero de matricula do aluno.

- curso
  Armazena o curso ao qual o aluno pertence.

- disciplinas
  Lista que armazena as disciplinas em que o aluno esta matriculado.

Metodos:

- __init__(self, nome, cpf, matricula, curso)
  Metodo construtor da classe. Ele chama o construtor da classe Pessoa usando
  super() e depois inicializa matricula, curso e disciplinas.

- adicionar_disciplina(self, disciplina)
  Adiciona uma disciplina a lista de disciplinas do aluno. Antes de adicionar,
  verifica se a disciplina ainda nao esta na lista, evitando duplicidade.

- listar_disciplinas(self)
  Retorna uma string com o nome das disciplinas do aluno. Se o aluno nao tiver
  disciplinas, retorna a mensagem "Nenhuma disciplina matriculada.".

- __str__(self)
  Retorna uma representacao textual do aluno, exibindo nome, CPF, matricula e
  curso.

Exemplo de uso:

aluno = Aluno("Felipe Santos", "555.666.777-88", "2026001", "Sistemas de Informacao")
print(aluno)


6. CLASSE PROFESSOR

Arquivo: Professor.py

A classe Professor representa um professor do sistema academico.

Heranca:

Professor herda da classe Pessoa. Por isso, todo professor possui nome e cpf.

Atributos:

- nome
  Herdado de Pessoa.

- cpf
  Herdado de Pessoa.

- registro
  Armazena o registro profissional ou institucional do professor.

- area
  Armazena a area de atuacao do professor.

- disciplinas
  Lista que armazena as disciplinas atribuidas ao professor.

Metodos:

- __init__(self, nome, cpf, registro, area)
  Metodo construtor da classe. Ele chama o construtor da classe Pessoa usando
  super() e depois inicializa registro, area e disciplinas.

- adicionar_disciplina(self, disciplina)
  Adiciona uma disciplina a lista de disciplinas do professor. Antes de
  adicionar, verifica se a disciplina ainda nao esta na lista.

- listar_disciplinas(self)
  Retorna uma string com o nome das disciplinas do professor. Se o professor
  nao tiver disciplinas, retorna a mensagem "Nenhuma disciplina atribuida.".

- __str__(self)
  Retorna uma representacao textual do professor, exibindo nome, CPF, registro
  e area.

Exemplo de uso:

professor = Professor("Mariana Souza", "111.222.333-44", "PROF001", "Programacao")
print(professor)


7. CLASSE DISCIPLINA

Arquivo: Disciplina.py

A classe Disciplina representa uma disciplina academica.

Atributos:

- nome
  Armazena o nome da disciplina.

- codigo
  Armazena o codigo identificador da disciplina.

- carga_horaria
  Armazena a carga horaria da disciplina.

- professor
  Armazena o professor responsavel pela disciplina. Inicialmente pode ser None.

- alunos
  Lista que armazena os alunos matriculados na disciplina.

Metodos:

- __init__(self, nome, codigo, carga_horaria, professor=None)
  Metodo construtor da classe. Inicializa nome, codigo, carga_horaria,
  professor e alunos. Se um professor for informado na criacao da disciplina,
  o metodo definir_professor e chamado automaticamente.

- definir_professor(self, professor)
  Define o professor responsavel pela disciplina. Alem disso, tambem adiciona
  essa disciplina a lista de disciplinas do professor.

- matricular_aluno(self, aluno)
  Matricula um aluno na disciplina. O metodo adiciona o aluno a lista de alunos
  da disciplina e tambem adiciona a disciplina a lista de disciplinas do aluno.
  Antes disso, verifica se o aluno ainda nao esta matriculado, evitando
  duplicidade.

- listar_alunos(self)
  Retorna uma string com o nome dos alunos matriculados na disciplina. Se nao
  houver alunos, retorna a mensagem "Nenhum aluno matriculado.".

- __str__(self)
  Retorna uma representacao textual da disciplina, exibindo nome, codigo,
  carga horaria e professor.

Exemplo de uso:

disciplina = Disciplina("Programacao Orientada a Objetos", "POO101", 80, professor)
print(disciplina)


8. CLASSE SISTEMAACADEMICO

Arquivo: Sistema_academico.py

A classe SistemaAcademico centraliza o gerenciamento de alunos, professores e
disciplinas.

Atributos:

- alunos
  Lista que armazena todos os alunos cadastrados.

- professores
  Lista que armazena todos os professores cadastrados.

- disciplinas
  Lista que armazena todas as disciplinas cadastradas.

Metodos:

- __init__(self)
  Metodo construtor da classe. Inicializa as listas alunos, professores e
  disciplinas.

- cadastrar_aluno(self, aluno)
  Adiciona um objeto Aluno a lista de alunos do sistema.

- cadastrar_professor(self, professor)
  Adiciona um objeto Professor a lista de professores do sistema.

- cadastrar_disciplina(self, disciplina)
  Adiciona um objeto Disciplina a lista de disciplinas do sistema.

- buscar_aluno_por_matricula(self, matricula)
  Procura um aluno pela matricula. Se encontrar, retorna o objeto Aluno. Se nao
  encontrar, retorna None.

- buscar_professor_por_registro(self, registro)
  Procura um professor pelo registro. Se encontrar, retorna o objeto Professor.
  Se nao encontrar, retorna None.

- buscar_disciplina_por_codigo(self, codigo)
  Procura uma disciplina pelo codigo. Se encontrar, retorna o objeto
  Disciplina. Se nao encontrar, retorna None.

- matricular_aluno_em_disciplina(self, matricula, codigo_disciplina)
  Procura o aluno pela matricula e a disciplina pelo codigo. Se ambos forem
  encontrados, matricula o aluno na disciplina. Caso o aluno ou a disciplina
  nao existam, exibe uma mensagem de erro.

- listar_alunos(self)
  Exibe todos os alunos cadastrados e suas disciplinas.

- listar_professores(self)
  Exibe todos os professores cadastrados e suas disciplinas.

- listar_disciplinas(self)
  Exibe todas as disciplinas cadastradas e seus alunos matriculados.

- gerar_dados_relatorio(self)
  Organiza os dados de alunos, professores, disciplinas e matriculas em uma
  estrutura propria para relatorios.

- gerar_relatorio_json(self, caminho="relatorio_academico.json")
  Gera um arquivo JSON com os dados do sistema academico.

- gerar_relatorio_pdf(self, caminho="relatorio_academico.pdf")
  Gera um arquivo PDF com os dados do sistema academico.


9. FUNCAO MAIN

Arquivo: Sistema_academico.py

A funcao main e o ponto principal de execucao do programa.

Ela realiza os seguintes passos:

1. Cria um objeto SistemaAcademico.
2. Cria um professor.
3. Cria dois alunos.
4. Cria uma disciplina com professor definido.
5. Cadastra o professor no sistema.
6. Cadastra os alunos no sistema.
7. Cadastra a disciplina no sistema.
8. Matricula os alunos na disciplina.
9. Lista professores, alunos e disciplinas.


10. FLUXO DE EXECUCAO

O programa deve ser executado pelo arquivo Sistema_academico.py:

python3 Sistema_academico.py

Quando executado, o Python verifica esta parte do codigo:

if __name__ == "__main__":
    main()

Isso significa que a funcao main sera chamada somente quando o arquivo
Sistema_academico.py for executado diretamente.

- Criacao de classes.

- Uso de atributos.
- Uso de metodos.

14.2 Banco de dados

Foi adicionada persistencia em banco de dados.

O sistema foi preparado para usar MySQL com as seguintes tabelas:

- alunos
- professores
- disciplinas
- matriculas

O arquivo schema.sql cria o banco sistema_academico, as tabelas, os
relacionamentos e alguns dados iniciais.

Como o MySQL nao estava disponivel na maquina durante os testes, tambem foi
adicionado um modo automatico com SQLite local. Assim, a aplicacao continua
funcionando mesmo sem um servidor MySQL ativo.

Arquivo relacionado:

- db.py

Funcoes principais adicionadas em db.py:

- get_connection()
  Abre uma conexao com o banco em uso.

- fetch_all(query, params=None)
  Executa uma consulta e retorna varias linhas.

- fetch_one(query, params=None)
  Executa uma consulta e retorna uma unica linha.

- execute(query, params=None)
  Executa comandos de escrita, como INSERT, UPDATE e DELETE.

- database_label()
  Informa qual banco esta sendo usado pela aplicacao.

14.3 Remocao logica de matriculas

O botao Remover da tela de matriculas foi ajustado para nao apagar os dados
do banco.

Antes, a remocao usava DELETE. Agora, a matricula recebe:

- ativo = 0
- removido_em = data e hora da remocao

Com isso:

- A matricula removida deixa de aparecer na interface.
- O registro continua armazenado no banco de dados.
- Se o mesmo aluno for matriculado novamente na mesma disciplina, a matricula
  antiga e reativada.

14.4 Relatorios em JSON e PDF

Foi criada a geracao de relatorios em dois formatos:

- JSON
- PDF

No arquivo Sistema_academico.py foram adicionados os metodos:

- gerar_dados_relatorio(self)
  Monta uma estrutura com alunos, professores, disciplinas e matriculas.

- gerar_relatorio_json(self, caminho="relatorio_academico.json")
  Gera o arquivo relatorio_academico.json.

- gerar_relatorio_pdf(self, caminho="relatorio_academico.pdf")
  Gera o arquivo relatorio_academico.pdf.

Tambem foi criado o arquivo relatorios.py, com funcoes reutilizaveis:

- gerar_relatorio_json(dados, caminho=None)
  Converte os dados para JSON e, se um caminho for informado, grava o arquivo.

- gerar_relatorio_pdf(dados, caminho=None)
  Cria um PDF simples com os dados do sistema e, se um caminho for informado,
  grava o arquivo.

Na interface web, foram adicionadas as rotas:

- /relatorios/json
  Baixa o relatorio em JSON com os dados atuais do banco.

- /relatorios/pdf
  Baixa o relatorio em PDF com os dados atuais do banco.

No painel inicial foram adicionados botoes para baixar os relatorios.
